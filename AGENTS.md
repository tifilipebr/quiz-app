# AGENTS.md

## Project Overview

A React Router v7 SPA quiz application built with React 19, TypeScript, and Tailwind CSS v4.

- **Stack**: React 19 + TypeScript + Tailwind CSS v4
- **Framework**: React Router v7 (SPA mode, `ssr: false` in react-router.config.ts)
- **Build tool**: Vite 7
- **Package manager**: npm
- **Path alias**: `~/*` maps to `./app/*` (configured in tsconfig.json paths)

## Commands

| Command                         | Description                                                   |
| ------------------------------- | ------------------------------------------------------------- |
| `npm install`                   | Install dependencies                                          |
| `npm run dev`                   | Start dev server (hot reload)                                 |
| `npm run build`                 | Production build via `react-router build`                     |
| `npm run start`                 | Serve production build (requires build first)                 |
| `npm run typecheck`             | Run type checking (`react-router typegen && tsc`)             |
| `npm run typecheck -- --noEmit` | TypeScript check only (same as above, tsc already has noEmit) |
| `npm run lint`                  | Run ESLint (all rules)                                        |
| `npm run lint:fix`              | Run ESLint with auto-fix                                      |
| `npm run format`                | Check Prettier formatting                                     |
| `npm run format:fix`            | Auto-format all files with Prettier                           |

**No test framework exists** (vitest/jest/playwright not configured). The only verification commands are `typecheck`, `lint`, and `format`. Always run `npm run typecheck && npm run lint` before committing.

## Development Workflow

- `npm run dev` starts the Vite dev server with HMR via `react-router dev`.
- The app is a client-side SPA (no SSR). All rendering happens in the browser.
- Use `~/` path alias for imports within `app/` (e.g., `import { QuizData } from "~/hooks/QuizData"`).
- Route files live in `app/routes/` and are registered in `app/routes.ts`.
- Styling uses Tailwind utility classes exclusively â€” no CSS modules, no styled-components.

## Code Style Guidelines

### Imports

- **Order**: External libraries first (react, react-router), then internal `~/` alias imports, then relative `./+types/` typegen imports.
- **Type-only imports**: Always use `import type { ... }` for type-only imports (enforced by `verbatimModuleSyntax: true` in tsconfig).
- **Path alias**: Use `~/` for all cross-directory imports within `app/`. Use relative imports only for same-directory typegen (e.g., `./+types/home`).
- **Side-effect imports**: CSS imports like `import "./app.css"` are allowed and placed after other imports.

### Exports

- **Route pages**: `export default function ComponentName` (Home, QuizRoute, App in root.tsx).
- **Shared components**: `export function ComponentName` for plain functions, `export const ComponentName: React.FC<Props>` for functional components.
- **Interfaces/types**: `export interface IInterfaceName` â€” named exports, always prefixed with `I`.

### Naming

- **Files**: PascalCase for components (`Question.tsx`, `QuizContent.tsx`), kebab-case for config (`react-router.config.ts`).
- **Components**: PascalCase.
- **Functions/variables**: camelCase.
- **Interfaces**: PascalCase with `I` prefix (e.g., `IQuestion`, `IAnswer`, `IQuizData`, `IQuestionParams`).
- **CSS classes**: Tailwind utility classes only â€” inline in className, no custom CSS classes.

### TypeScript

- **Strict mode**: Enabled globally with `noUncheckedIndexedAccess`. All code must satisfy strict type checking.
- **Interfaces over types**: Use `interface` (with `I` prefix) rather than `type` aliases for object shapes.
- **React types**: Use `React.FC<Props>`, `React.ReactNode`, `React.Dispatch<React.SetStateAction<T>>`.
- **Route types**: Import from `./+types/<routeName>` (auto-generated by react-router typegen).
- **`satisfies` keyword**: Preferred over explicit type annotations for config exports.
- **Optional chaining**: Use `?.` for nullable access (e.g., `currentQuestion?.question`).
- **Non-null assertion**: Use `!` sparingly (e.g., `setSelectedAnswer!(option)` when guarded by check).

### React Patterns

- **Functional components only**: No class components.
- **Hooks**: Use `useState`, `useEffect`, `useCallback`, `useMemo` from React.
- **No state management library**: Use local `useState`/`useEffect` patterns (see QuizData.tsx for the custom hook pattern).
- **Props**: Define inline or use an interface type. Avoid `React.FC` children typing â€” use `{ children: ReactNode }` or `{ children: React.ReactNode }`.
- **Event handlers**: Inline arrow functions or named functions. Use `event.preventDefault()` in click handlers on list items.
- **Conditional rendering**: Ternary `condition ? <A /> : <B />` and short-circuit `{condition && <Component />}`.

### Error Handling

- **Route-level**: Use `ErrorBoundary` export in `root.tsx` â€” handles `isRouteErrorResponse` (404, status errors) and dev-time stack traces.
- **Component-level**: Try/catch blocks inside render/map for defensive rendering, with `console.log(error)` fallback.
- **No global error boundary libraries**: Only the built-in React Router error boundary.

### Styling

- Tailwind CSS v4 with `@import "tailwindcss"` in `app.css`.
- All styles are inline utility classes â€” never create custom `.css` files or CSS modules.
- Common patterns: `flex flex-col`, `justify-center items-center`, `p-*`, `bg-*`, `border-*`, `rounded-*`, `text-* font-*`, `space-y-*`.

### State Management

- **Local state**: `useState` for component state.
- **Custom hooks**: Encapsulate related state + logic in a custom hook (e.g., `QuizData()` hook returns `IQuizData`).
- **Derived state**: `useMemo` for computed values.
- **Side effects**: `useEffect` with cleanup via returned `clearTimeout`/`clearInterval`.
- **Stable callbacks**: `useCallback` for functions passed as props or used in deps.

### Git & Branching

- Feature branches: `feat/<feature-name>`.
- Pull requests merged into `main`.
- Commit messages: imperative mood, lowercase (e.g., "add style to some components").
- No conventional commits enforced.

### Docker

- Multi-stage Docker build with `node:20-alpine`.
- Build: `docker build -t quiz-app .`
- Run: `docker run -p 3000:3000 quiz-app` (app listens on port 3000 by default).
- `.env` files are gitignored â€” no env vars currently needed.

### ESLint (enforced by `eslint.config.js`)

- **Config**: Flat config (`eslint.config.js`), ESLint 9 + `typescript-eslint` v8 strict type-checked.
- **Naming convention**: Interfaces must have `I` prefix + PascalCase; type aliases, classes, enums in PascalCase; functions in camelCase or PascalCase; variables in camelCase, PascalCase, or UPPER_CASE.
- **Import order**: Builtins â†’ externals (react first, alphabetized) â†’ `~/` internals â†’ parent/sibling â†’ index. Blank line between groups enforced.
- **Type imports**: `import type` required for type-only imports (`consistent-type-imports: error` with `fixStyle: inline-type-imports`).
- **No `any`**: `no-explicit-any: error`. No unused vars (`no-unused-vars: error`, allow `_` prefix).
- **Boolean expressions**: Must be explicit for numbers (e.g., `count > 0` not `if (count)`). Strings and nullable objects allowed as truthy checks.
- **Optional chain & nullish coalescing**: Preferred (`prefer-optional-chain`, `prefer-nullish-coalescing`).
- **No confusing void**: Warns on returning `clearTimeout`/`setTimeout` from arrow functions â€” use `{ }` braces.
- **React Hooks**: Full recommended ruleset (`react-hooks/exhaustive-deps`, `rules-of-hooks`).
- **React Refresh**: Warns on files with mixed component/non-component exports.

### Prettier (enforced by `.prettierrc`)
